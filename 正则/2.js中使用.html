支持正则的，string对象的方法

search ---> str.search(reg or toreg) ===> -1 index

  1. 类似indexOf,没匹配到时返回-1,匹配到时返回第一个匹配时的index
  2. 不同于类似indexOf,不能用于数组

match ---> str.search(reg or toreg) ===> null [描述] []

  1. 没匹配到时返回null
  2. 没g标识时，返回一个数组['匹配中1','匹配中2'...index:'第一个匹配中时的index',input:'原始字符串',groups:...]
  3. 有g标识时，返回一个数组['匹配中1','匹配中2']

replace ---> str.replace(str or reg, str or func) ===> str

  1. 第一个参数为str时，作为检索直接量文本，不会转为正则
  2. 第二个参数为str时，可以使用一些特殊变量，代替匹配中的字符串
    变量名 代表值
    $$ $
    $& 匹配中的字符
    $` 匹配中左边
    $' 匹配中右边
    $n 第n个括号匹配中
  3. 第二个参数为func时，可以return回想要格式的文本，函数参数(match,p1,p2,p3,offest,string)如下
    参数名 代表值
    match 匹配中的字符,对应$&
    p1,p2,...pn 第n个括号匹配中,对应$1,$2...
    offest 匹配中相对原字符串的偏移量
    string 原str

正则对象的方法

test ---> reg.test(str)

  1. 没匹配中的时候返回false
  2. 匹配中的时候返回true

exec ---> reg.exec(str) ===> null [描述]

  1. 类似match方法没g标识时，返回['匹配中1','匹配中2'...index:'第一个匹配中时的index',input:'原始字符串',groups:...]
  2. 有g和没g，返回值没差。但是有g的话，可以实现‘链式’执行。即reg有一个lastIndex属性。 有g匹配后，lastIndex设置为匹配中文本最后一个字符的下一个位置。没有匹配中时返回null。可用while循环遍历 没g匹配。lastIndex始终为0
